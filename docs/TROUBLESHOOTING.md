# Automatic-bootstrap troubleshooting

Start with `codex-inter-agent host status`, then follow the first matching branch.

For installation failures, run `INSTALL.cmd -DryRun` first. Add `-Json` by invoking `scripts\install-plugin.ps1` directly when machine-readable step/error output is useful. The dry run checks prerequisites and marketplace ownership but makes no changes.

| Symptom/code                             | Diagnose                                                                             | Safe action                                                                                                                                                    |
| ---------------------------------------- | ------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Desktop app detected, public CLI missing | The GUI reports that the packaged binary is private; `where codex` is empty          | Leave the official CLI option checked. Do not browse to or copy `WindowsApps\...\resources\codex.exe`; the wizard installs the compatible public CLI.          |
| macOS blocks the downloaded app          | Finder says the developer cannot be verified                                         | Confirm the Actions run and SHA-256, then Control-click → Open or use Privacy & Security → Open Anyway. Prefer the distinct notarized artifact when available. |
| macOS app cannot find Node/npm           | Finder-launched apps have a smaller `PATH` than Terminal                             | Install Node.js 22.11+/npm 10.9+ in a standard location or choose source diagnostics. The backend also probes the user's login shell; do not run it with sudo. |
| macOS reports private Codex executable   | The selected path is inside `Codex.app/Contents` or an editor extension              | Select a standalone public CLI or leave official CLI installation enabled. Do not copy an app-private executable.                                              |
| macOS artifact says unsigned             | Artifact name ends in `-unsigned`; `codesign` is ad hoc                              | This is expected for ordinary CI. It is tested but not Apple-notarized; only `-notarized.zip` from the protected passing job claims notarization.              |
| Installer cannot find Node/npm           | Read the failed prerequisite and `-DryRun` output                                    | Install Node.js 22.11+ and npm 10.9+, reopen the wizard, and retry.                                                                                            |
| Codex CLI version incompatible           | Compare `codex --version` with `generated/codex/manifest.json`                       | Use the GUI's official CLI option or select a matching standalone CLI. Do not bypass protocol compatibility validation.                                        |
| Invalid Codex data directory             | Check the GUI path and whether it exists                                             | Select the existing directory used by the desktop app, normally `%USERPROFILE%\.codex`; the wizard does not persistently rewrite `CODEX_HOME`.                 |
| Marketplace already configured elsewhere | `codex plugin marketplace list`; compare the `codex-inter-agent-local` root          | Do not overwrite it. Use that checkout's installer, or explicitly remove the known old marketplace before installing this checkout.                            |
| npm prefix/permission failure            | `npm.cmd prefix --global`; inspect the failed install step                           | Configure a writable current-user npm prefix. Do not rerun the whole wizard as administrator merely to bypass an unexplained ownership problem.                |
| Plugin not listed                        | `codex plugin marketplace list` and `codex plugin list`                              | Rerun `INSTALL.cmd`, then completely restart Codex or open a new task.                                                                                         |
| Installed CLI version mismatch           | `codex-inter-agent --version`; compare `package.json`                                | Rerun the same-checkout installer. If the repository moved, resolve the marketplace collision first.                                                           |
| MCP did not start                        | Inspect Codex MCP/plugin state and protected `host.log`                              | Confirm the plugin runtime exists and `BRIDGE_AGENT_ID` is set in trusted process configuration. Do not put it in tool input.                                  |
| Tools not exposed                        | Check caller registration/status/owner binding                                       | Register or `adopt-owner` the caller through the shared owner. Unknown, inactive, or unbound callers fail before tools are registered.                         |
| `HOST_LOCK_TIMEOUT`                      | `host status`; inspect descriptor/lock age and logged PIDs                           | Wait for the bounded launcher. Do not delete a live lock or start a second host.                                                                               |
| Stale descriptor/lock                    | `host status` reports stale; verify recorded processes                               | Use `host start` to run authenticated stale recovery. Never hand-edit the descriptor.                                                                          |
| `HOST_AUTH_FAILED` / wrong token         | Verify every process uses the same protected data/token paths                        | Stop the known owner and correct configuration. Never paste or rotate a live token through a prompt.                                                           |
| `HOST_INCOMPATIBLE`                      | Compare bridge, protocol, installation, database, transport, and app-server versions | Stop explicitly, install one synchronized release, then restart. Do not launch alongside it.                                                                   |
| Hidden child failed                      | Inspect redacted `host.log`, `host status`, and app-server version                   | Run the production build, correct the reported dependency/config issue, then start again.                                                                      |
| `UNSUPPORTED_THREAD_OWNER`               | `show <agent-id>` owner fields and whether the history is open in desktop/IDE        | Close independent owners; verify idle through the shared owner; run confirmed `adopt-owner`. Official desktop targets remain unsupported without an adapter.   |
| Recipient busy                           | Message status, live thread status, recipient lease                                  | Let the unrelated turn finish. The queue must not steer it.                                                                                                    |
| Pending after timeout                    | `get_request_status <message-id>` and `health` unfinished counts                     | Continue polling the same ID. Reconciliation uses the original client message ID.                                                                              |
| `RECIPIENT_CONTEXT_EXHAUSTED`            | Recipient transcript/context                                                         | Compact only through a supported operator workflow, then explicitly retry. Never auto-fork or silently rebind.                                                 |
| Approval denied                          | Recipient requested command/file/MCP elevation                                       | Reframe without the side effect or perform it in a separately human-authorized workflow. Peer messages do not grant approval.                                  |
| Schema drift                             | `npm.cmd run schema:check`                                                           | Regenerate against the installed Codex build, inspect protocol/adapters, and rerun the full gate.                                                              |

The installer may print npm warnings while still succeeding; only a nonzero command exit is a failure. On macOS, run `./INSTALL.command --dry-run --json` for a no-write diagnostic plan. If Windows reports a persistent file lock during plugin rebuild, close tools scanning the repository and retry. The builder retries transient `EBUSY`/`EPERM` replacement briefly but does not hide a persistent lock or permission error.

If status is ambiguous, do not use `--force` first. Preserve `host.log`, run `health`, record the signed descriptor fields without the separate token, and stop normally. Use `host stop --force` only when the operator intentionally accepts interruption of active deliveries.

For database corruption, stop all bridge processes, preserve the database/WAL/SHM set for diagnosis, and restore a verified hot backup with the matching package. Never edit `schema_migrations` or a Codex internal history database.
