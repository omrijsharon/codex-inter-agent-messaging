# Phase 0 Protocol Surface

Authority: experimental schemas generated by `codex-cli 0.144.0-alpha.4` under `spikes/phase0/generated`.

| Required capability | Generated evidence | Result |
|---|---|---|
| Initialize request | `app-server-schema/ClientRequest.json` contains `initialize` | Present |
| Initialized notification | `app-server-schema/ClientNotification.json` contains `initialized` | Present |
| Thread discovery | `ClientRequest.json` contains both `thread/list` and `thread/search` | Present |
| Thread read | `ClientRequest.json` contains `thread/read` | Present |
| Thread resume | `ClientRequest.json` contains `thread/resume` | Present |
| Turn start | `ClientRequest.json` contains `turn/start` | Present |
| Turn interrupt | `ClientRequest.json` contains `turn/interrupt` | Present |
| Completed item event | `ServerNotification.json` contains `item/completed` | Present |
| Completed turn event | `ServerNotification.json` contains `turn/completed` | Present |
| Idempotent user message ID | `v2/TurnStartParams.json` exposes `clientUserMessageId` | Present |
| Dynamic tools on new threads | `v2/ThreadStartParams.json` exposes `dynamicTools` | Present |
| Dynamic tools on resumed threads | `v2/ThreadResumeParams.json` does not expose `dynamicTools` | Absent as expected |
| Live thread state | `Thread` in `v2/ThreadResumeResponse.json` requires `status` | Present |
| Thread status variants | `notLoaded`, `idle`, `systemError`, and `active` | Present |
| Active waiting flags | `waitingOnApproval` and `waitingOnUserInput` | Present |
| Thread status notifications | `ServerNotification.json` exposes `thread/status/changed`; the payload carries `threadId` and `status` | Present |
| Terminal turn states | `completed`, `interrupted`, `failed`, and `inProgress` | Present |

Run the executable check with:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File spikes\phase0\verify-schema.ps1
```

The live thread status is usable for Phase 0 busy-state checks, but app-server remains authoritative: a `turn/start` rejection must still be handled as a possible busy condition because external activity can race with a prior status read.
